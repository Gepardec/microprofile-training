Open-Tracing Basic (Basics)

Open-Tracing Tracer

Instead of using the provided instrumentation, we can provide our own spans or enrich the current active one.

We will provide spans for all of the JAX-RS requests/responses to the training backend via JAX-RS ContainerRequestFilter and ContainerResponseFilter implementations.

Your task:

Implement the com.gepardec.training.microprofile.basic.opentracing.RestRequestFilter#filter and use the current active span and enrich it with the following tags:

  1. http-uri is the uri of the request
  2. http-method the http method of the request
  3. http-header-<NAME> all of the set headers of the request

Implement the com.gepardec.training.microprofile.basic.opentracing.RestResponseFilter#filter methods and create a new span with the operation name <CLASS_NAME> which contains the following tags:

  1. http-status-code is the http status code of the response

Your traces are visible in the logs and of course in the JaegerUI

Use the ContainerRequestContext, ContainerResponseContext instances provided to the filter methods, which will contain all the data you need.
Take your time to investigate the Tracer API, which will tell you how to get the current active span or how to create a new one.
