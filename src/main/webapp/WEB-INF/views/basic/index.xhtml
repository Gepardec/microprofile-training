Microprofile Basics

Microprofile Config facilitates the configuration of an application. With the following examples, you will configure this application in several ways and see how you can use the configured values. In two examples, these configured values will be mapped and converted into specific target types or even aggregated into a property class. Furthermore, you will handle empty configurations which, by default, lead to exceptions.

The specification for Microprofile Config.

Microprofile Health aims to provide a standardized way to expose information about the applications current liveness and readiness conditions cumulated as a health status. This Information is predicated to be utilized in container and cloud environments for orchestration, as well as for monitoring and alerting purposes independent of the environment.

TL;DR: /health endpoint so k8s can detect unhealthy pods

In the examples you will:
  • take a look at at the /health endpoint
  • correct faulty healthchecks
